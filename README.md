# Игротека вожатого

Десктопное приложение на Python + Tkinter + SQLite для учёта игр и инвентаря в детском лагере.

## Возможности
- Каталог игр (CRUD, поиск, фильтры по типу и возрасту)
- Несколько типов реквизита на одну игру с указанием количества
- Реальное бронирование инвентаря: при «Начать игру» количество резервируется,
  при «Завершить игру» — возвращается
- Защита от запуска игры, если инвентаря не хватает
- Управление складом: добавление, изменение общего количества, удаление
- Вкладка «Активные игры»
- Статистика: популярные игры, активность вожатых, полный журнал
- Встроенная справка

## Структура проекта
```
main.py        — GUI (Tkinter)
database.py    — работа с SQLite, схема и тестовые данные
igroteka.db    — файл БД (создаётся автоматически при первом запуске)
```

## Требования
- Python 3.8+ (tkinter и sqlite3 входят в стандартную поставку)
- Никаких дополнительных пакетов ставить не нужно

Если Python не установлен:
1. Скачайте установщик с https://www.python.org/downloads/
2. **Обязательно** поставьте галку «Add Python to PATH» при установке
3. Проверьте: в PowerShell `python --version` должно вывести версию

## Как запустить
В PyCharm: откройте папку проекта, правой кнопкой на `main.py` → Run.

Из терминала (находясь в папке проекта):
```
python main.py
```

При первом запуске создастся `igroteka.db` с примерными играми и инвентарём,
чтобы было что показать.

## База данных
- `Inventory(id, name, total_quantity, available_quantity)` — склад
- `Games(id, name, type, age_group, inventory_required)` — игры
- `Game_Inventory(game_id, inventory_id, quantity_per_game)` — связь many-to-many
- `Activity_Log(id, game_id, counselor_name, start_time, end_time)` — журнал
- `Session_Inventory(log_id, inventory_id, quantity_used)` — что именно
  зарезервировано в каждом сеансе (нужно для корректного возврата)

Внешние ключи включены (`PRAGMA foreign_keys = ON`), удаление каскадно.
